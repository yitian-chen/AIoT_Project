心模块一：智能课表管理
这是用户与系统交互最高频的入口，用于将“计划”下发给硬件。
1. 可视化周课表
功能描述：以“周视图”展示周一到周日的课程/行程。
交互：点击具体的“星期+时间段”格子，弹出添加/编辑窗口。
数据项：
课程名称（如：高等数学、去图书馆）
时间（星期几、开始时间）
地点（这是关键，需要关联导航坐标）
2. 地图选点
功能描述：在添加课程时，不能只输入文字，必须输入经纬度供ESP32导航。
实现：调用微信小程序的 <map> 组件或 wx.chooseLocation 接口。
逻辑：用户在地图上打点 -> 获取经纬度 -> 绑定到该课程。
3. 一键同步
功能描述：编辑好一周的课表后，点击“同步到设备”。
逻辑：将本地数组打包成 JSON，通过华为云 API 下发指令给 ESP32。
核心模块二：安全监护中心
这是系统的“被动触发”模块，用于处理紧急情况。
1. 紧急联系人设置
功能描述：允许用户输入或修改接收报警邮件的邮箱地址。
逻辑：将此邮箱地址通过云端下发给 ESP32 存储，或者小程序端直接调用云函数发送邮件（建议前者，即 ESP32 直接发邮件）。
2. 实时报警弹窗
功能描述：当 ESP32 检测到摔倒并上传数据到华为云后，小程序需通过 WebSocket 或 轮询机制 接收消息。
表现：屏幕弹出红色全屏警告，播放警报音，显示“检测到摔倒！”。
3. 事故定位导航
功能描述：在报警页面直接显示事故发生地的地图。
逻辑：解析 ESP32 上传的经纬度，调用腾讯地图（微信内置）显示位置，并提供“一键导航前往”按钮（方便救援）。
核心模块三：骑行数据看板
这是系统的“数据展示”模块，用于生成日报。
1. 每日骑行概览
功能描述：展示当天的骑行成果。
数据项：
总里程（km）
总时长（min）
平均速度（km/h）
消耗卡路里（可选，通过算法估算）
2. 骑行轨迹回放
功能描述：在地图上画出当天的骑行路线。
逻辑：如果 ESP32 实时上传了轨迹点，小程序可以用 <polyline> 在地图上描绘出路径。
3. 生成分享海报
功能描述：点击“生成日报”，生成一张精美的图片。
内容：背景图 + 核心数据大字 + 二维码。
用途：方便用户保存到相册并发朋友圈。
️ 核心模块四：设备状态监控
这是系统的“状态栏”，让用户知道设备是否在线。
1. 设备连接状态
显示：在线 / 离线 / 信号强度。
逻辑：监听华为云的设备影子或心跳包。
2. 实时位置追踪
功能描述：在小程序地图上实时显示 ESP32 设备当前的经纬度图标。
用途：防丢防盗，或者查看设备是否已到达目的地。

开发建议
地图组件：微信小程序使用的是腾讯地图，坐标体系是 GCJ-02。而 GPS 模块通常输出 WGS-84。在显示位置时，可能需要进行坐标转换，否则会有几百米的偏移。
订阅消息：为了实现“摔倒后手机收到通知”，你需要利用微信的“订阅消息”接口。当用户点击“同步课表”时，顺便请求一次发送通知的权限。
UI框架：建议使用 Vant Weapp 或 TDesign 组件库，可以快速搭建出漂亮的表单和仪表盘，不用自己手写 CSS。
